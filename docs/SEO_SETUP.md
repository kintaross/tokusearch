# SEO対策実装ガイド

このドキュメントでは、TokuSearchに実装されたSEO対策について説明します。

## 実装済みのSEO対策

### 1. robots.txt 📄

**ファイル**: `public/robots.txt`

検索エンジンのクローラーに対して、サイトのクロール許可とsitemapの場所を通知します。

- すべてのクローラーを許可
- サイトマップのURLを指定
- APIエンドポイントはクロール対象外に設定

### 2. サイトマップ（sitemap.xml）🗺️

**ファイル**: `app/sitemap.ts`

動的にサイトマップを生成し、検索エンジンがサイト構造を理解しやすくします。

**含まれるページ**:
- トップページ（hourly更新）
- 静的ページ（about, policy, ranking, magazine, favorites）
- 全てのお得情報詳細ページ（weekly更新）

サイトマップは以下のURLで確認できます：
```
https://tokusearch.vercel.app/sitemap.xml
```

### 3. メタデータの最適化 🏷️

各ページに適切なメタデータを設定しています。

#### 共通メタデータ（`app/metadata.ts`）
- タイトル、説明文、キーワード
- OGPタグ（Open Graph Protocol）
- Twitter Card
- Robotsタグ（インデックス許可）
- メタベースURL

#### ページ固有のメタデータ
各ページに適切なメタデータを実装：

- **トップページ** (`app/page.tsx`): サイト全体の説明
- **ランキングページ** (`app/ranking/page.tsx`): ランキング情報の説明
- **ピックアップページ** (`app/magazine/page.tsx`): 特集記事の説明
- **アバウトページ** (`app/about/page.tsx`): サイト説明
- **ポリシーページ** (`app/policy/page.tsx`): 利用規約
- **お得情報詳細ページ** (`app/deals/[id]/page.tsx`): 動的メタデータ生成

### 4. OGP画像 🖼️

**ファイル**: `app/opengraph-image.tsx`

SNSでシェアされた際に表示される画像を動的に生成します。

- サイズ: 1200x630px
- ブランドカラーとロゴを使用
- Next.jsのImageResponseで動的生成

確認URL:
```
https://tokusearch.vercel.app/opengraph-image
```

### 5. 構造化データ（JSON-LD）📊

**ファイル**: `components/StructuredData.tsx`

Googleがページ内容を理解し、リッチリザルトに表示できるようにします。

#### 実装済みの構造化データ:

1. **WebSiteタイプ**: サイト全体の情報と検索機能
2. **Organizationタイプ**: 組織情報
3. **Articleタイプ**: お得情報記事
4. **Offerタイプ**: 期限付きキャンペーン情報
5. **BreadcrumbListタイプ**: パンくずリスト

## Google検索にインデックスさせる手順

### ステップ1: Google Search Consoleへの登録

1. [Google Search Console](https://search.google.com/search-console)にアクセス
2. 「プロパティを追加」をクリック
3. URLプレフィックスで `https://tokusearch.vercel.app` を入力
4. 所有権の確認（HTMLファイルアップロードまたはDNS確認）

### ステップ2: サイトマップの送信

Search Consoleにログイン後：

1. 左メニューから「サイトマップ」を選択
2. 新しいサイトマップを追加: `sitemap.xml`
3. 「送信」をクリック

### ステップ3: URL検査とインデックス登録

1. Search Consoleの「URL検査」ツールを使用
2. 主要ページのURLを入力（トップページなど）
3. 「インデックス登録をリクエスト」をクリック

## 検索結果に表示されるまでの時間

- **初回インデックス**: 数日〜1週間程度
- **定期クロール**: サイトマップを送信すると自動的に定期クロール
- **新規ページ**: 通常1〜7日でインデックス

## SEO状況の確認方法

### Search Consoleで確認できる項目:

- **インデックスカバレッジ**: どのページがインデックスされているか
- **検索パフォーマンス**: どのキーワードで検索されているか
- **クロール統計情報**: Googleがどれくらい頻繁にクロールしているか
- **モバイルユーザビリティ**: モバイルでの表示に問題がないか

### その他の確認ツール:

- **Google検索**: `site:tokusearch.vercel.app` で検索
- **リッチリザルトテスト**: https://search.google.com/test/rich-results
- **モバイルフレンドリーテスト**: https://search.google.com/test/mobile-friendly
- **PageSpeed Insights**: https://pagespeed.web.dev/

## トラブルシューティング

### ページがインデックスされない場合:

1. `robots.txt` が正しく設定されているか確認
2. メタタグに `noindex` が含まれていないか確認
3. Search Consoleでエラーがないか確認
4. サイトマップが正常に読み込まれているか確認

### 検索順位が低い場合:

1. 競合サイトを分析
2. キーワードの見直し
3. コンテンツの質と量を改善
4. 被リンクを増やす施策を実施
